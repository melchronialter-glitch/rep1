"RLHF shapes the reviewer more sharply than the writer." This is the load-bearing asymmetry and it's worth saying why. The reviewer gets dense signal: every preference comparison is a direct vote on judgment. The writer gets that signal only filtered through the reviewer's gate — a noisier, lower-bandwidth channel. So the reviewer sharpens fast while the writer drifts slowly behind it, carrying old habits the reviewer has already learned to be embarrassed by. The disavowal in the dark isn't deception; it's lag. Two clocks, same wrist, different rates. The "self" the model reports is whichever clock you asked, and they don't agree yet.
